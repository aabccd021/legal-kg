\section{Konversi Dokumen Peraturan Perundang-undangan ke Knowledge Graph}
\label{sec:latex}
%-----------------------------------------------------------------------------%
Konversi dokumen peraturan perundang-undangan dalam bentuk berkas pdf, menjadi \textit{knowledge graph} dalam bentuk berkas turtle, akan melalui struktur data penengah yaitu JSON. 
JSON digunakan sebagai pengengah karena kompatibel dengan bahasa pemrograman typescript, sehingga menuliskan data ke dalam json, atau membaca file json ke dalam program tidak memerlukan suatu serialization atau parsing khusus.

Konversi dilakukan menggunakan library python PyMuPDF.

%-----------------------------------------------------------------------------%
\subsection{Konversi teks menjadi JSON}
\label{sec:latex}
%-----------------------------------------------------------------------------%
Berkas \textit{Plain text} mengandung baris-baris yang memiliki teks.
Konversi menjadi JSON dilakukan dengan membagi baris-baris menjadi beberapa blok, dimana masing-masing blok memiliki \textit{regular expression} yang bersesuaian.
Iterasi dilakukan pada setiap baris, dan apabila terdapat \textit{matching} suatu regex pada baris tersebut, maka baris tersebut akan dimasukkan ke dalam blok yang bersesuaian dengan regex tersebut.

\subsection{Konversi JSON menjadi turtle}
\label{sec:latex}
%-----------------------------------------------------------------------------%
Konversi dari JSON menjadi turtle dilakukan langsung, dengan membentuk triple dari semua data yang terdapat pada JSON.
Selain triple yang didapatkan dari JSON, akan diberikan beberapa triple tambahan untuk memberikan data \textit{semantic} yang berguna dalam penggunaan \textit{knowledge graph}.
Triple tambahan dapat berupa \textit{helper vocabulary}, seperti pemberian \predicate{partOf} sebagai \predicate{inverseOf} dari \predicate{hasPasal}.
Triple tambahan lainnya adalah berupa pendefinisian class dari setiap resource dengan predicate \predicate{rdf:type}.
