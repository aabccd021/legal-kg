  \item Visualisasi

%--------------------------------------------------------------------------------------------------%
\subsection{SPARQL Query}
\label{subsec:sparq-query}
%--------------------------------------------------------------------------------------------------%

Dengan menggunakan SPARQL server seperi Apache Jena Fuseki, KG yang dibuat dengan Lex2KG dapat
dianalisis dengan menjalankan SPARQL query. SPARQL query memiliki fitur operasi \textit{join},
\textit{union}, \textit{optional}, dan \textit{negation pattern}. SPARQL juga menyediakan query
analisis yang lengkap seperti \textit{filter}, \textit{aggregate}, dan \textit{property path}.
Dengan menggunakan SPARQL query, salah satu \textit{competency question} ``Tampilkan semua peraturan
serta peraturan yang ditimbangnya.'' dapat dijawab dengan melakukan query \lst~\ref{lst:query-menimbang}
yang akan memberikan output \tab~\ref{tab:output-query-menimbang}.

\begin{listing}[H]
  \begin{minted}[frame=single, fontsize=\footnotesize]{sparql}
PREFIX o: <https://example.org/lex2kg/ontology/>
PREFIX uu: <https://example.org/lex2kg/uu/>

SELECT ?penimbang ?ditimbang
WHERE {
  ?penimbang o:menimbang ?menimbang .
  ?menimbangText o:bagianDari* ?menimbang .
  ?menimbangText o:merujuk ?ditimbang .
  ?ditimbang a o:Peraturan
}
ORDER BY DESC(?penimbang) DESC(?ditimbang)
LIMIT 10
  \end{minted}
  \caption{SPARQL query untuk ``Tampilkan semua peraturan serta peraturan yang ditimbangnya.''}
  \label{lst:query-menimbang}
\end{listing}

\begin{table}
  \centering
  \begin{tabular}{|l|l|} \hline
    ?penimbang & ?ditimbang \\\hline \hline
    uu:2020/8  & uu:2018/12 \\\hline
    uu:2020/6  & uu:2020/2  \\\hline
    uu:2020/6  & uu:2015/1  \\\hline
    uu:2020/6  & uu:2014/1  \\\hline
    uu:2020/2  & uu:2020/1  \\\hline
  \end{tabular}
  \caption{Output Query \lst~\ref{lst:query-menimbang} }
  \label{tab:output-query-menimbang}
\end{table}

\subsection{Visualisasi KG}
\label{subsec:visualisasi-kg}
%--------------------------------------------------------------------------------------------------%

% // TODO: Ksh 3 yg jenisnya beda2
% // TODO: analisis insight yg didapat apa?

Visualisasi meningkatkan visibilitas dan \textit{explainability} KG. Pada KG dapat dilakukan
visualisasi \textit{node} dan \textit{relation} antara \textit{node}, atau visualisasi statistik
dari KG tersebut. \lst~\ref{lst:vizkg} merupakan kode untuk melakukan visualisasi ``Tampilkan 5 peraturan
dengan komponen terbanyak beserta jumlah komponennya'' menggunakan \textit{visualization library}
VizKG\footnote{\url{https://pypi.org/project/VizKG/}} dengan output \pic~\ref{fig:vizkg}.

\begin{listing}[H]
  \begin{minted}[frame=single, breaklines, fontsize=\footnotesize]{python}
import VizKG.visualize as VKG

sparql_query = """
PREFIX o: <https://example.org/lex2kg/ontology/>

SELECT ?docName (COUNT(?component) as ?componentCount)
WHERE {
  ?doc a o:Peraturan .
  ?component o:bagianDari* ?doc .
  BIND(REPLACE(REPLACE(STR(?doc), "https://example.org/lex2kg/", ""), "/", "-") as ?docName) .
}
GROUP BY ?docName
ORDER BY DESC(?componentCount)
LIMIT 5
"""

url = "http://127.0.0.1:3030/lex2kg/sparql"
bubble = VKG(sparql_query=sparql_query, sparql_service_url=url, chart='BarChart')
bubble.plot()
  \end{minted}
  \caption{Visualisasi ``Tampilkan 5 peraturan dengan komponen terbanyak beserta jumlah komponennya'' menggunakan VizKG}
  \label{lst:vizkg}
\end{listing}
